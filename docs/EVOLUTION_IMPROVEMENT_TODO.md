## 一、命令简化

### TODO 1.1: 增强 `sa evolve` 为一体化命令 ✅ 已决策

**当前问题：**
- 用户需要理解 `import` 和 `evolve` 两个命令的区别
- 用户不知道先用哪个命令
- 技能不存在于数据库时，evolve 直接报错

**改进方案：**
```bash
# evolve 成为一体化入口
sa evolve <skill>        # 查找 + 导入(如需) + 进化 + 报告

# 可选参数
sa evolve <skill> -v     # 详细模式
sa evolve <skill> --dry-run  # 仅预览，不应用
```

**智能行为：**

| 场景 | 行为 |
|-----|------|
| 技能在数据库中 | 直接进化，输出结果 |
| 技能在 OpenClaw 但未导入 | 自动导入 + 进化 |
| 技能不存在 | 报错 + 推荐相似技能 |
| 技能已是最新 | 提示"无需更新" |

**底层命令 (保留，高级用户)：**
- `sa import <skill>` - 仅导入，不进化
- `sa adapt <skill>` - 仅进化，别名

**实现要点：**
- [x] 决策：保留 `evolve` 命令名
- [x] 决策：自动导入未在数据库的 OpenClaw 技能
- [x] 决策：添加 `--dry-run` 预览模式
- [x] 实现智能查找逻辑
- [x] 实现自动导入流程
- [x] 实现简洁输出格式
- [x] 实现 `--dry-run` 模式

**优先级：** ⭐⭐⭐ 高 ✅ 已完成
